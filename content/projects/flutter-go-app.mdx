## Problem

Many projects have attractive frontends but weak backends, or vice versa. Clients need products where both user experience and backend functionality are well-integrated and reliable, especially in mobile applications that depend on remote services.

## Approach

I developed a complete mobile application using Flutter for the frontend and Go for the backend, with clearly defined API contracts between both layers. The frontend consumes RESTful services from the backend to retrieve and send data, while the backend provides the business logic and storage necessary.

## Outcome

The result is a functional mobile application that directly connects a polished user experience with a robust backend. This demonstrates capacity to work across the entire technology stack, from the user-facing interface to the services processing data behind the scenes.
